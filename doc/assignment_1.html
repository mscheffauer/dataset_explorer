Class BankingSystem (3P)

An instance of the class BankingSystem represents a bank. A BankingSystem handles bank accounts and provides functionality for creating and removing accounts, as well as for adding, retrieving, and transferring money (by calling the respective methods of the affected bank account/s). Furthermore, it can display the information of a bank account by calling the __str__ method of that account.

_account_id int The internal account ID, initialized with 0.
_bic str BIC code of the Bank.
_country str The country as ISO Alpha-2 code.
_bank_code str The bank-specific code.

These attributes have neither a getter nor a setter!

Required Method: __init__

  • Functionality:
    The __init__ creates a BankingSystem object.

  • Parameters:

    Name Type Description
    bic str The BIC for the bank account.
    country str The country as ISO Alpha-2 code. This is needed for the creation of the IBAN.
    bank_code str The bank-specific code (each bank has a specific code).

Required non-public Method: _checksum

  • Functionality:
    This method calculates the checksum for an user account. Each IBAN contains a checksum, which is derived from the account number, the country code and the bank code. The calculation works as follows:

    1. The country ISO Alpha-2 code has to be converted to a number. Each letter in the alphabet has a corresponding number. The counting starts at 10. This means A = 10, B = 11, C = 12, … Z = 35. For example, AT equals 1029, DE equals 1314.
      Hint: In Python, you can convert a letter to its unicode number. This can be done with ord(<letter>), eg: ord("A") would give 65, ord("B") would give 66.
    2. Then, two 0s are added to the country’s number representation. For Austria, this would be 102900, and for Germany 131400.
    3. Now, we can calculate a checknumber. This is done by concatenating the bank code, the account number, and the country’s number representation. For example, an Austrian bank code of 20815, an account number of 00042347891, and AT as country would result in the following number: 2081500042347891102900.
    4. The checksum is now calculated by the following formula: 98 - (checknumber modulo 97). Hint: modulo in Python is done with the % operator.
      For example, this would be: 98 - (2081500012347891102900 modulo 97) = 55. Thus, the checksum of this account would be 55.
    5. Ultimately, convert the checksum to a string. Since a checksum can result in a digit smaller than 10, you need to zero-pad this value to 2 digits. For example, a checksum of 2 would result in '02'.
      Hint: you can use format strings to do this: f"{checksum:02}" would pad the checksum with zeroes to 2 digits.

  • Positional arguments:

    Name Type Description
    account_number str The user-specific account number.

  • Return:

    Type Description
    str The checksum as a string.

Required Method: create_user_account

  • Functionality:
    This method should create a new user account and add the user to _accounts. First, check if the user already has an account. If that is the case, raise a ValueError with the following message: "An account with this user already exists". Else, increase the initial account number (_account_id) by 1. Calculate the checksum with the _checksum method. Now create the IBAN as follows: <Alpha-2 country code> + <checksum> + <bank_code> + <account_number>.
    The account_number mentioned above is the value of the _account_id zero-padded to 11 digits. For example, an _account_id of 1 would result in '00000000001'.
    Hint: you can use format strings to do this: f"{_account_id:011}" would pad _account_id with zeroes to 11 digits.
    For an account with the same values as in the example of the checksum method the IBAN would be AT552081500042347891. Now create a new BankAccount instance and add it to the _accounts dictionary with the user.

  • Positional arguments:

    Name Type Description
    user User User for whom an account should be created

Required Method: add_money

  • Functionality:
    This method should add money to the account given by the user. First, check if the user already has an account, if not, raise a ValueError with the following message: "This account does not exist". Use the add_money method from the BankAccount instance.
    Hint: You can create your own non public method which checks if an user already has an account since you will need this functionality more than once.

  • Positional arguments:

    Name Type Description
    user User User for whom money is to be added.
    amount float Amount of money to add.

Required Method: retrieve_money

  • Functionality:
    This method should retrieve money to the account given by the user. First, check if the user already has an account, if not, raise a ValueError with the following message: "This account does not exist". Use the retrieve_money method from the BankAccount instance.

  • Positional arguments:

    Name Type Description
    user User User for whom money is to be retrieved.
    amount float Amount of money to retrieve.

Required Method: transfer_money

  • Functionality:
    This method should transfer money from one account to another account. First, check if both users have an account, if not, raise a ValueError with the following message: "This account does not exist". Use the retrieve_money and add_money method from the BankAccount instance.

  • Positional arguments:

    Name Type Description

Required Method: account_info

    Hint: use the __str__ method from BankAccount.
